# HELP REQUEST - Persistent FUNCTION_INVOCATION_FAILED for /api/execute-outreach

## Issue
The `/api/execute-outreach` serverless function on Vercel is consistently failing with a `FUNCTION_INVOCATION_FAILED` error. This error indicates a critical crash within the Vercel runtime environment of the function, preventing it from executing application logic or returning a proper response.

## Debugging Steps Taken
I have performed extensive debugging to isolate the cause of this issue:

1.  **`generate_outreach.py` cURL Command Escaping:** Initially, the `execute_outreach_curl.sh` script (generated by `generate_outreach.py`) was producing malformed `curl` commands due to incorrect payload escaping. This was fixed by switching to a "here string" approach (`--data-binary @- <<EOF...EOF`), ensuring the JSON payload is correctly passed to `curl`.
2.  **`lib/logger.js` Isolation:** Suspecting issues with the custom logging utility, I temporarily replaced all calls to `logError` and `logInfo` with direct `console.error` and `console.log` statements within `api/execute-outreach.js`. The `FUNCTION_INVOCATION_FAILED` persisted, indicating `lib/logger.js` was not the root cause.
3.  **Request Body Parsing (`micro`'s `json` utility):** Identified that direct access to `req.body` in Vercel's `micro`-based functions often requires explicit JSON parsing. This was corrected by importing and using `const { emails } = await json(req);` to ensure the request payload was properly processed. The `FUNCTION_INVOCATION_FAILED` persisted.
4.  **SendGrid API Key Initialization Error Handling:** Added a `try/catch` block around `sgMail.setApiKey(sendgridApiKey);` to gracefully handle potential synchronous errors during API key validation. The `FUNCTION_INVOCATION_FAILED` persisted.
5.  **Complete SendGrid Bypass:** As a last resort, I completely commented out all SendGrid-related code (module import, API key setup, and email sending calls) within `sendEmails`, making the function return a dummy success. Even with SendGrid entirely removed from the execution path, the `FUNCTION_INVOCATION_FAILED` still occurred.

## Conclusion
The persistence of the `FUNCTION_INVOCATION_FAILED` error through these comprehensive debugging steps strongly suggests that the crash is occurring at a very low level within the Vercel serverless function's runtime environment, *before* the application's JavaScript code can fully execute or provide meaningful diagnostic output through `console.log`/`console.error`. This type of error is typically unrecoverable at the application level.
